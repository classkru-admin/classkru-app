# แผน report.html — ใหม่ทั้งหมด

> ไฟล์เดิม report.html ทิ้งทิ้ง → สร้างใหม่ตาม spec นี้

---

## โครงสร้างหลัก

```
report.html
├── Level 1 — หน้าเลือกห้อง (default)
└── Level 2 — เข้าห้อง → 4 Tab
    ├── วันนี้
    ├── สัปดาห์
    ├── เทอมนี้
    └── สรุปรวม
```

---

## Level 1 — หน้าเลือกห้อง

- Search bar (ค้นหาวิชา/ห้อง)
- Filter chip ตามวิชา (ดึงจาก sections จริง)
- Summary bar: ห้องทั้งหมด / เฉลี่ย% / ห้องน่ากังวล
- การ์ดแต่ละห้อง:
  - dot สีตาม level (ประถม=เหลือง, ม.ต้น=เขียว)
  - ชื่อวิชา + ห้อง + จำนวนคน
  - % attendance + progress bar
  - badge "เสี่ยง X" (ถ้ามี)
  - chevron →
- แบ่ง group ตามระดับ (ม.ต้น / ประถม)

---

## Level 2 — Tab วันนี้

- topbar: ← ชื่อวิชา + ห้อง
- summary card: มา / สาย / ขาด / ลา + progress bar + %
- section "ต้องติดตาม": แสดงเฉพาะคนผิดปกติ (ขาด/สาย/ลา)
- footer: "X คนที่เหลือมาเรียนปกติ"

---

## Level 2 — Tab สัปดาห์

- summary card: ช่วงวันที่ + "สอน X คาบ" + มา/สาย/ขาด/ลา + %เฉลี่ย
- แนวโน้มรายคาบ: bar chart ตามวัน/คาบที่สอนจริง (ไม่ใช่จันทร์-ศุกร์ตายตัว)
- นักเรียนที่ต้องติดตาม: เฉพาะคนมีปัญหาสัปดาห์นี้
- insight: 3 bullet คำนวณจาก logic ง่ายๆ ไม่ใช้ AI

---

## Level 2 — Tab เทอมนี้

- summary card: ช่วงเทอม + สอนแล้ว X คาบ + เฉลี่ย%
- legend: มา/สาย/ขาด/ลา
- รายชื่อนักเรียน (เฉพาะคนมีปัญหาก่อน):
  - avatar + ชื่อ + เลขที่ + %มา
  - dot pattern รายคาบ (สีตามสถานะ)
  - tag: ปกติ / เริ่มเสี่ยง / ต้องติดตาม
- footer: "X คนที่เหลืออยู่ในเกณฑ์ปกติ"

---

## Level 2 — Tab สรุปรวม

- health card: % ใหญ่ + label + health tag (สุขภาพดี/น่ากังวล)
- metric grid 2×2: ขาดรวม / สายรวม / นักเรียนเสี่ยง / คาบที่สอน
- insight card: 3 จุด (เสี่ยง / แนวโน้ม / ภาพรวม)
- รายชื่อเสี่ยง: avatar + ชื่อ + % + tag

---

## Tag Rules (คำนวณอัตโนมัติ)

| เงื่อนไข | Tag |
|---|---|
| มา ≥ 80% | ปกติ |
| มา 70–79% | เริ่มเสี่ยง |
| มา < 70% | ต้องติดตาม |

---

## Navigation ที่ต้องแก้พร้อมกัน

- ทุกหน้า sidebar/bottom nav: เปลี่ยน "ภาพรวม" → "รายงาน" ชี้ไป report.html
- ตัด "นักเรียน" ออกจาก sidebar (MVP)
- overview.html — ไม่ใช้แล้ว (legacy)

## Data Source

- ดึงข้อมูลจาก CKService (Supabase) ทั้งหมด
- ไม่ใช้ ck-data.js
- API ที่ต้องใช้: getSections, getAttendance, getStudents
