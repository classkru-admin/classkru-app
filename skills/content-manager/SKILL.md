# Content Manager — จัดการไฟล์/เนื้อหา resources

สกิลนี้คุมให้คลังสื่อของ Curriculum Copilot เป็นระเบียบ ค้นเจอ และพร้อมขายเป็นแพ็กเกจตามตัวชี้วัด

## โครงสร้างโฟลเดอร์มาตรฐาน

ยึดตามโครงข้อมูลของโปรเจกต์เสมอ:
```
Subject/
└─ Grade/
   └─ Standard/            (เช่น ว_1.3)
      └─ Indicator/        (เช่น ว_1.3_ม3-1)
         ├─ 01_lesson_plan.docx
         ├─ 02_worksheet.docx
         ├─ 03_quiz.docx
         ├─ 04_answer_key.docx
         ├─ 05_rubric.docx
         └─ ...
```

## ระบบตั้งชื่อไฟล์ (naming convention)

ตามแบบที่โปรเจกต์ใช้อยู่ใน `curriculum-copilot/`:
```
{ลำดับ2หลัก}_{ประเภท}_{รหัสตัวชี้วัดแบบย่อ}.{ext}
ตัวอย่าง: 02_worksheets_v111m31.docx  →  worksheet ของ ว 1.1.1 ม.3/1
```
กฎ:
- ขึ้นต้นด้วยเลขลำดับ 2 หลัก เพื่อให้เรียงตามขั้นการสอน
- รหัสตัวชี้วัดต่อท้ายเสมอ เพื่อให้ค้น/จับคู่ได้
- ใช้ตัวพิมพ์เล็ก คั่นด้วย `_` ห้ามเว้นวรรค/อักขระไทยในชื่อไฟล์ (กันปัญหา path)

## หน้าที่หลักของสกิล

1. **จัดระเบียบ** — ย้ายไฟล์กระจัดกระจายเข้าโฟลเดอร์ตัวชี้วัดที่ถูกต้อง
2. **ตรวจครบ/ขาด** — เช็คว่าตัวชี้วัดแต่ละตัวมี resources ครบ 10 รายการหรือยัง รายงานช่องว่าง
3. **ตรวจซ้ำ** — หาไฟล์ซ้ำ/เวอร์ชันเก่า (เช่น โฟลเดอร์ `*-backup-*`) แล้วเสนอให้รวบ
4. **ทำดัชนี (manifest)** — สร้าง/อัปเดต index ของคลังสื่อ (เช่น `indicators_m3.md` หรือไฟล์ JSON) ให้ระบบค้นหาตามตัวชี้วัดใช้ได้
5. **แพ็กเกจ** — รวมไฟล์ของตัวชี้วัดหนึ่งเป็นชุดพร้อมขาย (Lesson Plan + Worksheet + Quiz + Answer Key) ตาม monetization Phase 1 (49–99 บาท/ตัวชี้วัด)

## Workflow มาตรฐาน

1. สแกนโฟลเดอร์ปัจจุบันก่อน (`find` / `ls`) ทำรายการสิ่งที่มี
2. จับคู่ไฟล์ → ตัวชี้วัด จากชื่อไฟล์/เนื้อหา
3. เสนอแผนการจัด (ย้ายอะไรไปไหน) **ให้ผู้ใช้ยืนยันก่อนย้ายจริง**
4. ดำเนินการ แล้วอัปเดต manifest/index
5. รายงานสรุป: ตัวชี้วัดไหนครบ ตัวไหนยังขาดอะไร

## ข้อควรระวัง

- **ห้ามลบไฟล์โดยไม่ยืนยัน** โดยเฉพาะโฟลเดอร์ `*-backup-*`
- เก็บไฟล์ต้นฉบับ (.docx) ไว้เสมอ แม้จะ export PDF แล้ว
- ก่อนรวบ backup ให้เทียบความต่างก่อน ไม่ทับของใหม่ด้วยของเก่า
- อ้างอิงโครงข้อมูลและตัวชี้วัดจริงเสมอ ไม่เดารหัสตัวชี้วัดเอง
